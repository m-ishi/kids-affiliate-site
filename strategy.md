## 5. 5-Layer Architecture

```
Layer 0: FOUNDATION
  メーカーDB / 製品DB / 季節カレンダー / 安全規制DB / 競合DB

Layer 1: RADAR (毎日)
  メーカー新商品 / おもちゃ大賞 / 安全リコール / 海外→日本展開
  Amazon/楽天ベストセラー変動 / 競合新記事 / 季節トリガー

Layer 2: STRATEGY (週次)
  priority = search_demand × (1 - competitor_coverage)
           × seasonal_urgency × product_availability × freshness_decay

Layer 3: PIPELINE
  Researcher → Writer(パパラボ) → Critic → Safety Auditor → SEO → Builder
  → HTML生成 + git push → Cloudflare自動デプロイ

Layer 4: PATROL (毎週日曜)
  価格変動 / 安全リコール / 季節記事年次更新 / 海外→日本展開
  コンテンツギャップ vs 競合(my-best, ouchi-iku)
```
